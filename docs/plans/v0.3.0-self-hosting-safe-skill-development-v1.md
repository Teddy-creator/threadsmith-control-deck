# v0.3.0 Self-hosting-safe Skill Development v1

## Goal

Keep Threadsmith self-hosting safe while developing Threadsmith itself: the stable installed `$threadsmith` controller must stay separate from repository skill/protocol source changes.

## Scope

In scope:

- Add explicit self-hosting guardrails for installed skill path vs repository skill source path.
- Reject unsafe self-hosting config where repo source points at the installed global Codex skill.
- Require installed-skill controller mode to declare an installed skill path.
- Expose runtime safety warnings so route decisions can explain why external adapters are blocked.
- Add focused tests proving unsafe self-hosting boundaries fall back to built-in mini protocols.

Out of scope:

- Modifying or installing `/home/user/.codex/skills/threadsmith/SKILL.md`.
- Real external skill invocation.
- Control Deck UI changes.
- Multi-provider automation.
- Native desktop shell.
- Hosted backend, embeddings, or RAG.
- Broad skill orchestrator redesign.

## Done When

- Domain schema rejects self-hosting config that allows global skill mutation.
- Domain schema rejects installed-skill mode without `installedSkillPath`.
- Domain schema rejects matching repository and installed skill paths.
- Domain schema rejects repo-source mode pointing at a global `.codex/skills` path.
- Runtime safety inspection returns protected installed skill paths and warnings.
- Runtime route resolution blocks external adapters when self-hosting boundaries are unsafe.
- Focused and package-level domain/runtime tests pass.

## Verification

```bash
npm run test --workspace @threadsmith/domain -- src/skillOrchestrator.test.ts
npm run test --workspace @threadsmith/runtime -- src/skillOrchestrator.test.ts src/miniProtocols.test.ts
npm run test --workspace @threadsmith/domain
npm run test --workspace @threadsmith/runtime
jq empty .threadsmith/*.json .threadsmith/context/*.json .threadsmith/context/role-packets/*.json
git diff --check
```

## Stop Condition

Stop when self-hosting boundary checks are implemented and verified without touching the installed global `$threadsmith` skill, UI, multi-provider automation, or real external skill execution.
