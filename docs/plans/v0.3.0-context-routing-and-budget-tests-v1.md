# v0.3.0 Context Routing / Budget Tests v1

## Goal

Make Threadsmith's context-routing contract testable: execution packets should point each role to the compact current context packet and its role-specific packet, while budget checks should flag heavy middle sections before they become long-prompt drag.

## Scope

In scope:

- Add context packet and role packet references to generated execution packets.
- Keep role context routing role-specific instead of handing every role the full packet.
- Add budget warning coverage for heavy middle sections, based on the lost-in-the-middle risk.
- Add tests that prove adapter-backed routes still preserve mini protocol input and output semantics.
- Keep this slice test-first and runtime/orchestrator focused.

Out of scope:

- Control Deck UI changes.
- Real external skill invocation.
- Multi-provider automatic execution.
- Global `$threadsmith` installation.
- Native desktop shell.
- Hosted backend, embeddings, or RAG.
- Broad packet schema redesign.

## Done When

- Planner, executor, reviewer, verifier, closeout, and hygiene execution packets include `.threadsmith/context/current-packet.json`.
- Each role execution packet includes only its matching `.threadsmith/context/role-packets/<role>.json` role packet reference.
- Budget tests flag heavy middle sections with an explicit lost-in-the-middle warning.
- Role packet tests prove high-noise sections are omitted from roles that do not need them.
- Mini protocol tests prove an available external-adapter route still carries bounded input and output checklists from the fallback protocol.
- Focused runtime/orchestrator tests pass.

## Verification

```bash
npm run test --workspace @threadsmith/runtime -- src/contextBudget.test.ts src/roleContextPacket.test.ts src/miniProtocols.test.ts
npm run test --workspace @threadsmith/orchestrator -- src/rolePackets.test.ts
jq empty .threadsmith/*.json .threadsmith/context/*.json .threadsmith/context/role-packets/*.json
git diff --check
```

## Stop Condition

Stop when context routing and budget behavior are locked by focused tests. Do not continue into UI, real external skill execution, or multi-provider automation in this slice.
