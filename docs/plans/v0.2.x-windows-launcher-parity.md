# v0.2.x Windows Launcher Parity Implementation Plan

## Goal

Bring Windows startup ergonomics closer to the existing macOS `.command` launchers while keeping Threadsmith v0.2.x a local-first web control deck.

## Scope

- Add PowerShell launchers for the front door and project-specific entry.
- Keep npm startup as the universal fallback for macOS, Windows, and Linux.
- Update docs so Windows users do not need to reverse-engineer macOS commands.
- Add launcher verification that can run on macOS while syntax-checking PowerShell scripts.

## Non-goals

- No native Windows desktop app.
- No installer, code signing, auto-update, or `.exe` packaging.
- No v0.3.0 Skill Orchestrator or multi-provider behavior.
- No change to the stable macOS `.command` behavior except docs / verification references.

## Tasks

1. [x] Add `Launch-Threadsmith.ps1` with front-door, explicit project root, port, dry-run, and skip-open support.
2. [x] Add `Open-Threadsmith-App.ps1` as the Windows front-door shortcut.
3. [x] Update `package.json` verification scripts so macOS launcher checks and PowerShell syntax checks are separate and composed by `verify:launchers`.
4. [x] Update README and usage guide with Windows-specific commands.
5. [x] Run launcher dry-runs, JSON/package checks, and relevant tests.
