# v0.3.0 Stop Rules / Recovery Hardening v1

## Goal

Harden autopilot continuation decisions so Threadsmith refuses unsafe `continue`, `start`, or `resume` actions when committed truth and the latest phase-run state disagree.

## Scope

In scope:

- Detect accepted committed truth even when the latest phase run is still `running` or `paused`.
- Keep safe paused resume behavior only when committed truth has not already accepted the phase.
- Preserve the existing "accepted previous run + reset current truth" path that starts a new phase run.
- Improve continuation details so the operator knows whether to wait, reset, inspect artifacts, or resume.
- Add focused tests for continuation and existing stop-controller behavior.

Out of scope:

- Control Deck UI changes.
- Multi-provider execution.
- External skill invocation.
- Global `$threadsmith` installation.
- Repair cap changes.
- Role packet protocol instruction changes.

## Done When

- `decideAutopilotContinuation` returns `reset-needed` when accepted committed truth conflicts with running or paused phase-run state.
- Paused phase runs still return `resume` when committed truth is not accepted.
- Accepted previous runs still return `start` after current truth has been formally reset.
- Running phase runs still return `wait` when truth is not accepted.
- Stop-controller tests still prove risk pause, infra retry, repair, and loop-limit behavior.
- Package-level verification passes.

## Stop Condition

Stop when continuation decisions are safer and verified. Do not continue into Control Deck UI, multi-provider, external skills, or broader context-budget work in this slice.
