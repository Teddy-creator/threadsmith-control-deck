# v0.2.0 Release Hardening

## Goal

Prepare the public web release surface for Threadsmith v0.2.0 by making version, documentation, release checklist, and verification guidance internally consistent with the completed Context OS lane.

## Scope

- Align README release/status wording with the current v0.2.0 Context OS development line.
- Add v0.2.0 release checklist and release-note draft.
- Update public links that still point only to v0.1.1 when they should mention v0.2.0.
- Keep the current web app behavior and visual style unchanged.
- Keep `.threadsmith` truth aligned with this release-hardening phase.
- Run release-level local verification.

## Non-goals

- Native desktop shell.
- Multi-provider automation.
- New UI redesign.
- Hosted backend, embeddings, RAG, or model API calls.
- Changing the conductor/chat boundary.

## Done When

- README clearly explains that v0.2.0 is the current Context OS line and v0.1.1 remains the latest stable release until the tag is published.
- v0.2.0 release checklist exists and reflects the current verification commands.
- v0.2.0 release notes draft exists and highlights Context Packet, Repo Map, Evidence Summary, Role Packets, recovery, and `sync-context`.
- Version-related docs no longer make the public surface feel stuck on v0.1.1.
- Local release verification passes.

## Verification

```bash
npm run test
npm run build
npm run test:e2e
npm run verify:launchers
npm run smoke:self-host
git diff --check
jq empty .threadsmith/*.json .threadsmith/context/*.json
```

## Release Impact

After this slice, v0.2.0 should be ready for final publish prep: tag/release notes, optional screenshots, and GitHub Release creation.
