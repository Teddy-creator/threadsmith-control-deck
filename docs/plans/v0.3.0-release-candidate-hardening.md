# v0.3.0 Release-candidate Hardening

## Goal

Determine whether Threadsmith v0.3.0 is ready to become a release candidate using committed truth, docs/research consistency, package behavior, and fresh verification evidence.

## Scope

In scope:

- Check `.threadsmith` truth for stale or contradictory v0.3.0 state.
- Check README, docs, plans, research artifacts, scripts, package metadata, and release-facing text for stale or misleading claims.
- Fix narrow release-candidate inconsistencies that would confuse users or future contributors.
- Run available package and release verification commands locally.
- Record unavailable checks, residual risks, and the final ready/not-ready decision.

Out of scope:

- Publishing a GitHub Release or changing repository visibility.
- Modifying or installing `/home/user/.codex/skills/threadsmith/SKILL.md`.
- Implementing multi-provider automation.
- Adding real external skill invocation.
- Native desktop shell work.
- Hosted backend, embeddings, or RAG.
- Large UI redesign or new product scope.

## Done When

- Completed v0.3.0 slice truth is consistent across `.threadsmith` state, context packet, evidence summary, and closeouts.
- README/docs/plans/research artifacts do not contain release-blocking stale v0.2.0 or old-slice claims.
- Any narrow inconsistency found during hardening is fixed or recorded as a residual risk.
- Available package/release verification commands pass with fresh evidence.
- The phase records whether v0.3.0 is release-candidate ready and what should happen next.

## Verification

```bash
npm run test --workspaces --if-present
npm run build --workspaces --if-present
npm run verify:launchers
npm run smoke:self-host
npm run smoke:autopilot
npm run smoke:deck-route
jq empty .threadsmith/*.json .threadsmith/context/*.json .threadsmith/context/role-packets/*.json
git diff --check
```

`npm run smoke:deck-route` is a release gate and therefore uses an isolated temporary project plus the fake Codex fixture by default. Real Codex CLI bridge checks remain available as an explicit local diagnostic with `npm run smoke:deck-route -- --real <project-root>`.

## Stop Condition

Stop after a ready/not-ready release-candidate decision is backed by fresh evidence and written into `.threadsmith` truth. Do not publish a release, install the global skill, or expand into multi-provider work in this phase.
