# v0.3.0 Built-in Mini Protocols v1

## Goal

Turn the accepted Skill Orchestrator routing schema into concrete built-in mini protocol contracts.

This slice should let Threadsmith answer: "If no external skill is installed, what bounded instruction should this role receive next?"

## Scope

In scope:

- Define built-in mini protocol contracts for `brief`, `plan`, `debug`, `review`, `verify`, `closeout`, `handoff`, `recover`, and `research`.
- Each protocol must declare purpose, required inputs, required outputs, stop conditions, and continuation hints.
- Add runtime helpers that convert a route decision into a role-ready protocol instruction.
- Preserve route decision metadata such as selected source, adapter id, availability, and safety warnings.
- Add focused domain/runtime tests.

Out of scope:

- Calling external skills or CLIs.
- Persisting protocol runs.
- Implementing the autopilot role loop.
- Control Deck UI changes.
- Installing or mutating the global `$threadsmith` skill.

## Done When

- Domain schema validates the complete built-in protocol inventory.
- The protocol inventory includes all v1 mini protocols.
- Runtime instruction generation works for built-in-only configs.
- Runtime instruction generation preserves fallback and safety metadata when an adapter is stale, disabled, missing, or unsafe.
- Generated instructions include a stop condition and continuation hint so a later role loop can stop safely.
- New helpers are exported from domain/runtime indexes.
- Focused tests and package-level verification pass.

## Stop Condition

Stop after mini protocol contracts and route-to-instruction helpers are implemented and verified. Do not continue into autopilot execution in this slice.
