# External Skill Adapter v1 Implementation Plan

> **For agentic workers:** Execute task-by-task. Keep this slice focused on discovery, health checks, project routing config, and fallback route decisions. Do not implement true external skill execution in this slice.

**Goal:** Let Threadsmith automatically discover local Codex skills, infer their capabilities, surface health/safety status, and route workflow roles to safe adapters with built-in mini protocol fallback.

**Architecture:** Add a discovery layer that reads `SKILL.md` metadata from global and project skill directories, converts discovered skills into adapter candidates, scores capabilities from metadata and known aliases, then resolves route decisions with explicit health and safety explanations. Existing mini protocols remain the fallback kernel and existing route decisions remain the acceptance boundary.

**Tech Stack:** TypeScript, Zod schemas in `@threadsmith/domain`, pure runtime inference in `@threadsmith/runtime`, filesystem scanning in `@threadsmith/fs-bridge`, Vitest.

---

## Scope

In scope:

- Parse Codex skill `SKILL.md` frontmatter and body summary.
- Discover skills from configured roots such as `/Users/cloud/.codex/skills` and project-local `.codex/skills`.
- Infer capabilities: `brief`, `plan`, `debug`, `review`, `verify`, `closeout`, `handoff`, `recover`, `research`, `frontend`, `docs`, `implement`.
- Generate adapter declarations from discovered skills.
- Report health: `available`, `missing`, `stale`, `disabled`, `unsafe`.
- Route using discovered adapters and preserve fallback decisions.
- Add docs explaining that v1 discovers and routes; it does not directly invoke external skills as an autonomous plugin runtime.

Out of scope:

- Running arbitrary external skill code.
- Modifying or installing global skills.
- Multi-provider execution.
- UI redesign.
- Native desktop app work.

## Task 3: Filesystem Skill Scanner
