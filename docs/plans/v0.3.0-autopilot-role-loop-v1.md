# v0.3.0 Autopilot Role Loop v1

## Goal

Wire built-in mini protocol instructions into Threadsmith's existing serial autopilot phase runner.

The current phase runner can already launch planner, executor, reviewer, verifier, and closeout in sequence. This slice makes that loop protocol-guided: every role packet should carry the selected built-in mini protocol, its guardrails, stop condition, required inputs, required outputs, and continuation hint.

## Scope

In scope:

- Extend `ExecutionPacket` with optional mini protocol instruction metadata.
- Attach built-in mini protocol instructions to planner, executor, reviewer, verifier, closeout, and hygiene packets.
- Render protocol metadata into role prompts.
- Preserve existing stop-controller decisions, repair-loop behavior, pause records, and accepted flow.
- Add tests proving role packets and phase runner launches are protocol-guided.

Out of scope:

- External skill invocation.
- Multi-provider routing or automatic provider selection.
- Control Deck UI changes.
- Native desktop shell.
- Mutating `/home/user/.codex/skills/threadsmith`.
- Changing stop-controller semantics or repair loop caps.

## Done When

- `ExecutionPacket` can parse both legacy packets and protocol-guided packets.
- `buildPacketForRole` attaches the correct built-in protocol instruction for each role.
- `renderRolePrompt` exposes protocol inputs, outputs, guardrails, stop condition, continuation hint, and route metadata.
- `PhaseRunner` success-path tests prove launched packets include protocol instruction metadata.
- Existing repair, resume, and pause tests still pass.
- Verification evidence is written back to `.threadsmith/context/evidence-summary.json`.

## Stop Condition

Stop when autopilot role packets are protocol-guided and focused verification passes. Do not continue into stop-rule hardening, UI surfacing, or multi-provider automation in this slice.
