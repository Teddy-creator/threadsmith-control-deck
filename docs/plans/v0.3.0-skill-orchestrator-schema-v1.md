# v0.3.0 Skill Orchestrator Schema v1

## Goal

Define the first Threadsmith Skill Orchestrator schema and routing decision model without calling external skills yet.

This slice turns the v0.3.0 research plan into a typed foundation:

- mini protocol IDs
- external skill adapter declarations
- capability registry shape
- routing decision shape
- fallback policy
- safety constraints for self-hosting

## Why This Slice Comes First

Threadsmith already has provider routing, role packets, next-best-step logic, and command bridge surfaces. What it does not yet have is a first-class way to say:

- "For this role and task, use built-in `plan`."
- "An external `debug` skill exists, but it is optional."
- "The external adapter is missing or stale, so fallback to built-in `debug`."
- "Adapter output is only evidence/proposal, not direct truth mutation."
- "Repository skill source must not overwrite the installed global `$threadsmith` controller."

The schema must come before implementation so future executor work has a small contract instead of another large prompt.

## Scope

In scope:

- Add a domain-level schema for Skill Orchestrator configuration and routing decisions.
- Add runtime helpers that resolve a role/task into built-in mini protocol vs external adapter.
- Add tests for fallback behavior, adapter availability, freshness, and self-hosting safety flags.
- Export the new domain/runtime types from package indexes.
- Keep current `.threadsmith` truth aligned to this slice.

Out of scope:

- Actually invoking external skills.
- Installing or mutating `/home/user/.codex/skills/threadsmith`.
- UI changes in the Control Deck.
- Multi-provider automatic execution.
- Native desktop shell.
- Hosted backend, embeddings, or RAG.
- Broad refactors to existing provider routing or command bridge behavior.

## Schema Sketch

Candidate domain types:

```ts
type MiniProtocolId =
  | "brief"
  | "plan"
  | "debug"
  | "review"
  | "verify"
  | "closeout"
  | "handoff"
  | "recover"
  | "research";

type SkillCapability =
  | MiniProtocolId
  | "implement"
  | "frontend"
  | "docs";

type SkillAdapterAvailability =
  | "available"
  | "missing"
  | "stale"
  | "disabled"
  | "unsafe";

type SkillOrchestratorRouteSource =
  | "built-in"
  | "external-adapter"
  | "fallback";
```

Candidate config shape:

```ts
interface SkillAdapterDeclaration {
  id: string;
  label: string;
  capabilities: SkillCapability[];
  entry: {
    kind: "codex-skill" | "cli" | "human";
    ref: string;
  };
  fallbackProtocol: MiniProtocolId;
  safety: {
    canMutateCommittedTruth: boolean;
    canMutateGlobalSkill: boolean;
    forbiddenPaths: string[];
  };
}

interface SkillOrchestratorConfig {
  version: 1;
  builtInProtocols: MiniProtocolId[];
  adapters: SkillAdapterDeclaration[];
  defaultFallback: MiniProtocolId;
  selfHosting: {
    activeController: "installed-skill" | "repo-source";
    repositorySkillPath: string;
    installedSkillPath: string | null;
    allowGlobalSkillMutation: boolean;
  };
}
```

Candidate runtime decision shape:

```ts
interface SkillRouteDecision {
  role: PhaseOwner;
  requestedCapability: SkillCapability;
  selectedProtocol: MiniProtocolId;
  selectedAdapterId: string | null;
  source: SkillOrchestratorRouteSource;
  availability: SkillAdapterAvailability;
  reason: string;
  safetyWarnings: string[];
}
```

The executor may refine naming and exact shape, but the tests should preserve these behaviors.

## Done When

- Domain schema parses a minimal built-in-only config.
- Domain schema parses an optional external adapter declaration.
- Domain schema rejects unsafe self-hosting defaults that allow global `$threadsmith` mutation.
- Runtime route helper selects a built-in mini protocol when no adapter is configured.
- Runtime route helper selects an available external adapter when it satisfies the requested capability.
- Runtime route helper falls back to the configured mini protocol when the adapter is missing, stale, disabled, or unsafe.
- Runtime route helper reports safety warnings without mutating committed truth.
- New types are exported from domain/runtime package indexes.
- No Control Deck UI or global installed skill files are changed.

## Research Mapping

- SWE-agent: keep the agent-computer interface explicit by modeling route decisions and adapter entry points as typed objects.
- ReAct: route decisions should preserve an action/observation/evidence loop instead of allowing adapter output to directly accept work.
- MemGPT: separate committed config, runtime route decisions, context packets, and evidence as memory tiers.
- Lost in the Middle: keep route packet inputs small and capability-specific.
- AutoGen: role boundaries remain explicit; a planner route cannot act as verifier.
- LangGraph: route decisions should be stateful edges that can be tested and recovered.

## Stop Condition

Stop this slice once the schema and routing helper are implemented, focused tests pass, and acceptance is ready for review. Do not continue into UI surface, external invocation, or global skill installation in the same slice.
